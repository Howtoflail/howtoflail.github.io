Food delivery webapp

A fullstack webapp that was created using React for frontend and SpringBoot for backend.
This web application has Docker configured for continuous integration and uses SonarQube for code quality.

This application was made for a university project for the third semester and it's purpose was to get me familiarized with creating and configuring a solid REST API for the needs of a specific business/application, but also having a user-friendly frontend, so that the user can easily use the website.

The web application excels mostly at the backend, having a solid foundation for most of the needs of a food delivery application.

Features of the application

One of the most common, but most important features of food delivery applications, is to have the ability to create accounts because it is very convenient to just log back in the website every time and have all of your personal data and address saved and order food within seconds.

On this application, you need to create an account in order to use it.

All forms on the website are validated because every piece of data that interacts with the database is following a format declared by request classes in the backend.

Once logged in, a user can see all the restaurants on the home page.

Logging in is done through the use of authentification and authorization. The webapp uses JWT tokens to differentiate between roles.

Users can click on a restaurant in order to see what items they offer.

On the restaurant's page, the user can choose the items and the quantity that he wants to order, which is displayed in a basket that also shows the total calculated in real time.

The user can see all details of the order, including personal details, before confirming it.
